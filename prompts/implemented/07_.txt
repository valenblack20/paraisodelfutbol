3. DATABASE MIGRATIONS
==================================================

Create new migrations only.

Do not modify applied migrations.

Create:

005_create_admin_users.sql
006_create_admin_sessions.sql
007_create_login_attempts.sql

TABLE admin_users

- id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY
- email VARCHAR(190) NOT NULL UNIQUE
- password_hash VARCHAR(255) NOT NULL
- display_name VARCHAR(120) NOT NULL
- active BOOLEAN NOT NULL DEFAULT TRUE
- last_login_at TIMESTAMP NULL
- created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
- updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP

TABLE admin_sessions

- id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY
- admin_user_id BIGINT UNSIGNED NOT NULL
- token_hash CHAR(64) NOT NULL UNIQUE
- csrf_token_hash CHAR(64) NOT NULL
- expires_at TIMESTAMP NOT NULL
- last_seen_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
- created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP

Foreign key:

- admin_user_id -> admin_users.id
- ON DELETE CASCADE
- ON UPDATE RESTRICT

Indexes:

- admin_sessions(admin_user_id)
- admin_sessions(expires_at)

TABLE admin_login_attempts

- id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY
- email_hash CHAR(64) NOT NULL
- ip_hash CHAR(64) NOT NULL
- successful BOOLEAN NOT NULL DEFAULT FALSE
- attempted_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP

Indexes:

- (email_hash, attempted_at)
- (ip_hash, attempted_at)

Do not store raw session tokens.
Do not store raw CSRF tokens.
Do not store raw IP addresses in this table.

==================================================
6. SESSION POLICY
==================================================

Use opaque server-side sessions.

Cookie name:

paraiso_admin_session

Cookie requirements:

- HttpOnly
- Secure in production
- SameSite=Lax
- Path=/
- no JavaScript access
- explicit Max-Age or Expires
- recommended session duration: 8 hours
- rotate session after login
- delete cookie on logout

Do not use JWT.
Do not store sessions in localStorage.
Do not store the raw token in MySQL.

Session token:

- at least 32 random bytes;
- Base64URL or hex;
- store SHA-256 hash only.

==================================================
7. CSRF
==================================================

Implement CSRF protection for authenticated state-changing actions.

Requirements:

- generate one CSRF token per session;
- store only its hash;
- expose the raw token only to authenticated server-rendered forms or a protected endpoint;
- require it for POST, PUT, PATCH and DELETE;
- compare using timing-safe logic;
- reject invalid or missing tokens with 403;
- do not rely only on SameSite cookies.

Login may use an unauthenticated CSRF strategy or strict Origin/Referer validation plus a one-time token.

Document the chosen approach.

==================================================
8. LOGIN RATE LIMIT
==================================================

Protect login by email and IP hash.

Suggested rule:

- maximum 5 failed attempts within 15 minutes;
- temporary rejection with HTTP 429;
- generic message in Spanish;
- do not reveal whether the email exists;
- successful login resets or neutralizes the effective failure count;
- hash email and IP before persistence;
- use a server-side pepper from environment configuration.

Add:

AUTH_RATE_LIMIT_PEPPER

Never expose it.

Do not trust `X-Forwarded-For` unless proxy trust is explicitly configured.

Document deployment proxy requirements.
